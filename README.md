# Part 1: Data Audit, EDA & Business Understanding

## D2C Customer Churn Intelligence — Capstone Project

### Overview
This repository contains the complete data audit, exploratory data analysis (EDA), and business understanding work for a D2C personal-care brand's churn intelligence system. The goal is to understand the data, identify data-quality issues, and surface churn-risk patterns before any model is built.

## Setup Instructions

### 1. Clone the Repository
```bash
git clone <your-repo-url>
cd part1-data-audit-eda
```

### 2. Create a Virtual Environment (Recommended) Check you have python 3.11.x
### Selected python 3.11 as it has good support for AI ML
```bash
py -3.11 -m venv .venv
source .venv/bin/activate        # On Windows: .venv\Scripts\activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Launch the Notebook
```bash
jupyter notebook eda_audit.ipynb
```

Run all cells from top to bottom. Charts will be saved to `charts/` and summary outputs to `outputs/`.

## Findings Summary

- **Churn rate** in the labeled dataset is ~28–32% (class imbalance present)
- **Recency** is the strongest single churn predictor — customers with no order in 60+ days churn at 2.4× the base rate
- **Support ticket volume** (especially complaints/returns) strongly correlates with churn
- **Discount-dependent customers** show high churn after discount windows close
- **Low app engagement** (< 2 sessions/month) is a strong leading indicator

See `business_memo.md` for prioritized recommendations.
